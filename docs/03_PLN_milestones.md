## Phase 1 — Hello World + базовый Gemini запрос

**Цель:** минимальное рабочее приложение. Окно открывается, сообщение отправляется, ответ приходит.

### Что реализуем

```
[ Строка ввода                    ] [Отправить]
──────────────────────────────────────────────
Ответ от Gemini появляется здесь (полностью, без стриминга)
```

### Задачи

- [ ] **P1-01** Создать `go.mod`, подключить `fyne.io/fyne/v2` и `google.golang.org/genai`
- [ ] **P1-02** Главное окно: `Entry` + кнопка + `Label` для ответа
- [ ] **P1-03** Жёстко прописать модель `gemini-2.0-flash` (API key из переменной окружения `GEMINI_API_KEY`)
- [ ] **P1-04** Отправка запроса в горутине (UI не замерзает)
- [ ] **P1-05** Кнопка блокируется на время запроса (`sendBtn.Disable()` / `Enable()`)
- [ ] **P1-06** Базовая обработка ошибок: показать `dialog.ShowError()` если запрос упал
- [ ] **P1-07** `go run ./cmd/app/` запускается без ошибок на macOS

### Структура файлов после Phase 1

```
cmd/app/main.go          — точка входа, всё в одном файле (допустимо на Phase 1)
go.mod / go.sum
```

### Критерии готовности (Definition of Done)

- ✅ Окно открывается за < 2 секунды
- ✅ Ввести текст → нажать Enter или кнопку → получить ответ
- ✅ Во время запроса кнопка задизейблена, UI не зависает
- ✅ При пустом `GEMINI_API_KEY` — понятное сообщение об ошибке
- ✅ `go vet ./...` без warnings

## Phase 2 — Streaming + Markdown рендер

**Цель:** текст появляется постепенно (как в ChatGPT), Markdown форматируется корректно.

### Что реализуем

```
┌─────────────────────────────────────────────────────┐
│  👤 Вы                                              │
│  Объясни горутины                                   │
│  ─────────────────────────────────────────────────  │
│  ✦ Gemini                                           │
│  Горутина — это лёгкий поток...   ▌ (стримится)    │
│                                                     │
│  ## Пример кода                                     │
│  ```go                                              │
│  go func() { ... }()                               │
│  ```                                                │
└─────────────────────────────────────────────────────┘
```

### Задачи

- [ ] **P2-01** Рефакторинг: разбить `main.go` на пакеты согласно архитектуре из `02_DES`
  - `internal/provider/provider.go` — интерфейс
  - `internal/provider/gemini/gemini.go` — реализация
  - `internal/ui/fyne/` — весь UI
- [ ] **P2-02** Реализовать `provider.Stream()` → канал `chan string`
- [ ] **P2-03** `widget.NewRichTextFromMarkdown()` для отображения ответа
- [ ] **P2-04** При стриминге: `richText.ParseMarkdown(accumulated)` на каждый чанк
- [ ] **P2-05** `scroll.ScrollToBottom()` после каждого чанка
- [ ] **P2-06** Визуальный индикатор стриминга: мигающий курсор `▌` в конце текста
- [ ] **P2-07** Пузырьки сообщений (`bubble.go`): разделение user / assistant
- [ ] **P2-08** Многострочный `Entry` (Shift+Enter = новая строка, Enter = отправить)
- [ ] **P2-09** Кнопка "Стоп" (отмена через `context.Cancel()`) появляется во время стриминга

### Ловушки этой фазы

```
⚠️  ParseMarkdown вызывается из горутины — нужно убедиться что Refresh
    вызывается корректно (Fyne v2 допускает Refresh из горутины).

⚠️  Незакрытые MD-токены при стриминге ("**бол" без закрывающих **).
    Решение: ParseMarkdown(полный накопленный текст) — работает корректно.

⚠️  scroll.ScrollToBottom() при каждом чанке может мигать.
    Решение: вызывать не чаще чем раз в 50ms через time.Ticker.
```

### Критерии готовности

- ✅ Текст появляется постепенно (стриминг виден)
- ✅ `**bold**`, `` `code` ``, `# заголовок`, блоки кода — рендерятся корректно
- ✅ Кнопка "Стоп" отменяет генерацию через `ctx.Cancel()`
- ✅ Прокрутка следует за текстом автоматически
- ✅ Код в ответе отображается моноширинным шрифтом
- ✅ Пакеты разделены согласно архитектуре из Phase 1 Design

## Phase 3 — История чатов (SQLite)

**Цель:** чаты сохраняются между запусками. Левая панель со списком чатов.

### Что реализуем

```
┌──────────────┬──────────────────────────────────────┐
│ [+ Новый]    │  (история чата загружается)           │
│ ──────────── │                                       │
│ ✦ Объясни.. │                                       │
│ ✦ Помоги с. │                                       │
│ ✦ Напиши... │                                       │
│              │                                       │
│ [🗑 Удалить] │                                       │
└──────────────┴──────────────────────────────────────┘
```

### Задачи

- [ ] **P3-01** Подключить `modernc.org/sqlite` (pure Go, без CGo)
- [ ] **P3-02** Реализовать `internal/store/store.go` — интерфейс
- [ ] **P3-03** Реализовать `internal/store/sqlite.go` — SQLite с PRAGMA WAL
- [ ] **P3-04** При старте: создать `~/.config/gemini-chat/history.db` если не существует
- [ ] **P3-05** `sidebar.go` — `widget.List` с чатами, отсортированными по `updated_at DESC`
- [ ] **P3-06** Создать новый чат → `store.CreateChat()` → обновить sidebar
- [ ] **P3-07** Выбрать чат из sidebar → `store.LoadMessages()` → отобразить историю
- [ ] **P3-08** Каждое отправленное сообщение → `store.AddMessage()` сразу (до ответа)
- [ ] **P3-09** Каждый полученный ответ → `store.AddMessage()` после завершения стриминга
- [ ] **P3-10** Автозаголовок чата = первые 45 символов первого сообщения пользователя
- [ ] **P3-11** Кнопка "Удалить чат" → `dialog.ShowConfirm()` → `store.DeleteChat()`
- [ ] **P3-12** История передаётся в Gemini как контекст (`contents []*genai.Content`)

### Ловушки этой фазы

```
⚠️  SQLite и конкурентные записи: использовать один write-connection
    с SetMaxOpenConns(1) и отдельный read-connection.

⚠️  При загрузке большой истории (100+ сообщений) UI может тормозить.
    Решение: загружать последние 50 сообщений, остальные — по запросу.

⚠️  Gemini имеет лимит контекстного окна.
    Решение: передавать не более последних N сообщений (настраивается).
```

### Критерии готовности

- ✅ Закрыть и открыть приложение → история чатов сохранена
- ✅ Список чатов отсортирован по времени последнего сообщения
- ✅ Переключение между чатами работает корректно
- ✅ Gemini отвечает с учётом истории диалога (помнит контекст)
- ✅ Удаление чата удаляет и его сообщения (ON DELETE CASCADE)
- ✅ `history.db` создаётся автоматически при первом запуске

## Phase 4 — Настройки и Keychain

**Цель:** убрать API key из переменной окружения. Полноценный диалог настроек.

### Что реализуем

```
┌─────────────────────────────────────────────────────┐
│  ⚙  Настройки                                       │
│                                                     │
│  ✅ Ключ сохранён: AIza••••••••K9fQ                 │
│                                                     │
│  Новый API ключ:  [●●●●●●●●●●●●●●●●●●●●]           │
│  → Получить бесплатно на Google AI Studio           │
│  ✅ Ключ рабочий, доступно моделей: 12              │
│                                                     │
│  Модель: [ Gemini 2.0 Flash              ▼ ]        │
│  Контекст: 1 000 000 токенов | Бесплатный тир       │
│                                                     │
│  Тема: (●) Системная  ( ) Тёмная  ( ) Светлая       │
│                                                     │
│  [ 🗑 Удалить ключ ]                                │
│              [ Отмена ]      [ Сохранить ]          │
└─────────────────────────────────────────────────────┘
```

### Задачи

- [ ] **P4-01** Реализовать `internal/keystore/keyring.go` через `99designs/keyring`
- [ ] **P4-02** При первом запуске без ключа → автоматически открыть диалог настроек
- [ ] **P4-03** При запуске с ключом → тихая валидация в фоне (`ValidateKey()`)
- [ ] **P4-04** `settings.go` — диалог с полем ввода ключа (PasswordEntry)
- [ ] **P4-05** `OnChanged` поля ключа → дебаунс 500ms → `ValidateKey()` → `FetchModels()`
- [ ] **P4-06** Выпадающий список моделей заполняется динамически из API
- [ ] **P4-07** Показывать `DisplayName` в списке, использовать `ID` для запросов
- [ ] **P4-08** Кнопка "Удалить ключ" → `dialog.ShowConfirm()` → `keystore.Delete()`
- [ ] **P4-09** Маскировка ключа: `AIza••••••••K9fQ`
- [ ] **P4-10** Переключение темы: system / dark / light через `forcedVariant` паттерн
- [ ] **P4-11** Сохранение выбранной темы и модели в SQLite таблицу `config`
- [ ] **P4-12** Баннер в главном окне при невалидном ключе + кнопка "Настройки →"

### Ловушки этой фазы

```
⚠️  На macOS при первом обращении к Keychain — системный диалог разрешения.
    Решение: KeychainTrustApplication: true в конфиге keyring.

⚠️  FetchModels() занимает 1-3 секунды — не блокировать UI.
    Решение: горутина + modelSelect.Disable() во время загрузки.

⚠️  Дебаунс обязателен: без него FetchModels() вызывается на каждое нажатие клавиши.
    Решение: time.AfterFunc(500ms, ...) с отменой предыдущего таймера.
```

### Критерии готовности

- ✅ `GEMINI_API_KEY` env переменная больше не нужна
- ✅ Ключ хранится в macOS Keychain (виден в Keychain Access)
- ✅ Повторный запуск — ключ загружается автоматически
- ✅ Список моделей загружается из API при вводе ключа
- ✅ Смена темы применяется без перезапуска
- ✅ При удалении ключа → приложение переходит в состояние "ключ не задан"
