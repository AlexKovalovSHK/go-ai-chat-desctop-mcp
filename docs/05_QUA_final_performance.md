## P-03: CPU при стриминге

```
Метод: Activity Monitor → CPU % во время получения длинного ответа

Тест: "Напиши подробное эссе на 2000 слов о языке Go"
Длительность стриминга: ~60-90 сек

CPU замеры (каждые 10 сек):
  0 сек:  ___%
  10 сек: ___%
  20 сек: ___%
  30 сек: ___%
  Пик:    ___%

Цель: пик < 25% на M1/M2
Результат: ⬜

Если CPU > 40%:
  Причина A: ParseMarkdown() вызывается слишком часто
  Решение:   Throttle до 1 раза / 50ms через time.Ticker

  Причина B: scroll.ScrollToBottom() на каждый чанк
  Решение:   Вызывать не чаще чем ParseMarkdown()

  Причина C: messageBox.Refresh() перерисовывает весь VBox
  Решение:   Refresh только последнего пузырька (bubble.Refresh())
             вместо полного контейнера
```

## P-04: CPU в покое (Idle)

```
Метод: Оставить приложение открытым на 5 минут, не взаимодействуя
Записать CPU каждую минуту:

  1 мин:  ___%
  2 мин:  ___%
  3 мин:  ___%
  4 мин:  ___%
  5 мин:  ___%

Цель: < 1% стабильно
Результат: ⬜

Если CPU > 1% в покое:
  Причина A: Fyne делает постоянные перерисовки
  Диагностика: добавить логирование в Draw() методы виджетов
  Решение: убедиться что нет активных Ticker/Timer которые вызывают Refresh

  Причина B: горутина не завершилась после стриминга
  Диагностика: go tool pprof /debug/pprof/goroutine → смотреть количество
  Решение: проверить что все каналы закрываются через defer close()
```
